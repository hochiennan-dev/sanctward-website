Solutions · 解決方案

把舊架構的困境，換成零信任的解法

從 VPN、遠端存取到身份與合規——六個最常見的場景，Sanctward 一套平台一次補上。

取代 SSL VPN

從「開放整個內網」到「只開放被授權的應用」

傳統 SSL VPN 連上後，等於把整座內網對使用者敞開，而且看不見加密隧道裡究竟發生什麼事——這正是橫向移動與資料外洩最常見的破口。

困境
過度信任：連上即可橫向移動到任意內網主機。
看不見內容：加密隧道內的操作無法即時監控。
缺端點控管：不檢查設備是否合規就放行，帳密被盜即長驅直入。
Sanctward 解法
零信任 per-app 授權，後端位置隱藏，沒有「內網」可供橫移。
每次連線即時信任評分、持續再驗證，異常即降分或強制斷線。
端點 Agent 檢查設備姿態（OS／加密／EDR）合規才放行。
連線過程全程錄影存證，並即時建立 OCR 全文索引——事後可用關鍵字秒級回放、定位任何畫面。
取代遠端存取軟體 · 強化遠端協定

RDP / VNC / SSH / Telnet，全走一道受控閘道

在端點安裝第三方遠端軟體，等於在公司內開了一條 IT 看不到的常駐通道；而直接對外開放遠端連接埠，則是把攻擊面攤在網路上。兩者都讓「誰連進來、做了什麼」變成黑洞。

困境
端點常駐通道繞過企業管控，誰連入、做了什麼，看不到也查不到。
連線碼或帳密一旦外流即遭接管，無錄影、無稽核軌跡。
對外暴露 RDP／VNC／SSH／Telnet 連接埠，成為直接攻擊入口。
Sanctward 解法
四種協定全走 Sanctward 閘道，端點免裝第三方軟體、後端埠不對外。
全程會話錄影 + OCR 全文搜尋 + 指令稽核；受控帳密注入，維護人員不接觸明文密碼。
JIT 時限存取、到期零殘留；剪貼簿 DLP 與檔案傳輸 SHA-256 完整性驗證。
連線共享 · Session Shadow

同一個畫面，多人共視、即時接管、全程留痕

遠端協作、教育訓練、緊急處置與稽核監看，常常需要「看到同一個畫面」。Sanctward 讓管理者即時共享任何進行中的 RDP／VNC／SSH／Telnet 連線畫面，並能由「觀看」無縫切換為「接管」，過程全程錄影並即時產生可搜尋索引。

困境
遠端協助時各自連線、無法同畫面協作，出狀況難以即時介入。
緊急時需資深人員接手，卻無法即時接管同一個連線。
監看與教學過程沒有紀錄，事後無法回溯、缺乏稽核軌跡。
Sanctward 解法
Multi-Session Shadow：即時觀看任何進行中的連線，多人同時共視同一畫面。
一鍵由「觀看」切換為「接管」，由授權人員直接操作緊急處置。
全程錄影 + 即時 OCR 全文索引——誰觀看、誰接管、做了什麼，全部留痕可搜尋。
統一身份與 MFA

多帳號、多系統、多種 MFA，收斂成一次登入

每套系統各有帳號密碼、各自一套 MFA（甚至完全沒有），管理分散、稽核有盲區，員工離職時帳號更難以一次收回乾淨。

困境
帳號分散：員工每套系統各一組帳密，IT 重設與回收耗時又易遺漏。
MFA 不一致：有的 TOTP、有的簡訊、有的全無，體驗差又留風險。
舊系統不支援 SSO／MFA，長期游離在統一管控之外。
Sanctward 解法
表單代填／反向代理／OIDC・SAML 聯邦，9 成以上認證模式皆可納管，舊系統不改造也能接。
內建 IdP 提供一致 MFA（TOTP），對所有應用統一，並可依風險動態升級驗證。
AD／LDAP 同步，離職一處停用、全系統即時失效，條件式存取一致套用。

台灣資安法 · 個資法 · 金管會，一次對齊
